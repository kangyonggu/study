Tab

1. 초기 상태

- 지정된 하나의 Tab 활성화, 그 외에 Tab은 비활성화
- 활성화 된 Tab에 연결된 컨텐츠만 표시되고 그 외 컨텐츠는 비활성화


2. Tab 기능 구현 조건

- Tab 버튼 클릭 시 :
   > 비활성화 된 Tab 클릭 시 해당 Tab과 연결된 컨텐츠 활성화
   > 그 외 나머지 Tab과 연결된 컨텐츠들은 비활성화


3. Tab 기능 구현을 위한 스크립트 작성로직

- 전체 Tab박스 변수 지정(페이지 내 다른 Tab이 있을 경우 중복 방지)
- Tab 버튼 변수 지정
- 컨텐츠 변수 지정

- Tab 버튼을 클릭했을때 이벤트 작성
  > Tab 버튼과 컨텐츠를 순서에 맞게 연결
  > Tab 버튼 클릭 시 Tab 버튼 active 클래스 제거
  > 클릭한 Tab 버튼은 active 클래스 추가
  > 컨텐츠 active 클래스 제거
  > 클릭한 Tab 버튼과 연결된 컨텐츠에 active 클래스 추가


4. Tab안에 텍스트/이미지 외 다른 태그가 있을 경우

- iframe : iframe 태그로 영상 자동재생을 할 경우
  > 스크롤이 Tab에 도달하고 Tab이 활성화 되면 자동 재생 시작
  > 비활성화 된 Tab에 영상 재생 정지 후 초기화

- form : 입력한 정보값을 전송할 경우
  > 입력한 정보값을 전송하더라도 해당 Tab은 활성화 유지


5. 해당 페이지가 반응형이 아닌 적응형일 경우
- 만약 적응형 페이지에서 pc/mo 모두 기능이 동일할 경우
  > pc/mb 모두 공용 변수로 통제 가능

- 각각의 사이즈에서 새로고침해서 DOM 초기화나 재생성의 경우
  > 초기화 함수를 만들고 resize 이벤트로 재호출






아코디언

1. 초기 상태

- 아코디언 내 모든 컨텐츠 비활성화 


2. 아코디언 메뉴 클릭 시 해야 하는 일

- 비활성화 된 메뉴 클릭 할 시 해당 메뉴와 연결된 컨텐츠 활성화
- 비활성화 된 메뉴를 클릭 할 시 활성화 된 메뉴와 컨텐츠는 비활성화 처리
- 활성화 된 메뉴 클릭 시 연결된 컨텐츠 비활성화 처리


3. 접근성

- 메뉴에 aria-expanded = "false" / role ="tab" / aria-controls에는 컨텐츠에 기재된 id로 지정
- 컨텐츠에 role = "tabpanel" / hidden / aria-labelledby에는 메뉴에 기재된 id로 지정
- 활성화 시 메뉴에 aria-expanded = "true" 처리 / 컨텐츠에 hidden은 제거
- 키보드 접근성 보장(좌우 화살표 및 Tab키로 포커스 이동 / Enter,space키로 포커스 활성화)
- tabindex 속성으로 활성/비활성화 관리의 경우 활성화 시 tabindex = 0, 비활성화 시 tabindex = -1로 설정


4. 예외 및 확장 고려
   > 컨텐츠 활성화 시 자동 재생
   > 컨텐츠 비활성화 시 자동 재생 정지 후 초기화
